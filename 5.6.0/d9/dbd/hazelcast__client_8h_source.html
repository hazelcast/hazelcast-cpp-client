47/*
48 * You can use native C++ Client to connect to hazelcast nodes and make almost
49 * all operations that a node does. Different from nodes, clients do not hold
50 * data.
51 *
52 * Some of features of C++ Clients are:
53 * * Access to distributed data structures like IMap, IQueue, MultiMap, ITopic
54 * etc... For complete list see the classes extending DistributedObject
55 * * Access to transactional distributed data structures like TransactionalMap,
56 * TransactionalQueue etc...
57 * * Ability to add cluster listeners to a cluster and entry/item listeners to
58 * distributed data structures.
59 * @see MembershipListener, IMap#add_entry_listener , IQueue#add_item_listener
60 * etc .
61 * * C++ Client is smart by default, which means that it knows where the data is
62 * and asks directly to correct node. Note that you can turn this feature off (
63 * client_config#setSmart), if you don't want your clients to connect every
64 * node.
65 *
66 * Our C++ client is completely open source and the source code is freely
67 * available at https://github.com/hazelcast/hazelcast-cpp-client . Please feel
68 * free to contribute. You can join our community at
69 * https://hazelcastcommunity.slack.com/channels/cpp-client where you can find
70 * answers to your questions.
71 */
