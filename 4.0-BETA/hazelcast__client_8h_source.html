29  /*
30  * You can use native C++ Client to connect to hazelcast nodes and make almost all operations that a node does.
31  * Different from nodes, clients do not hold data.
32  *
33  * Some of features of C++ Clients are:
34  * * Access to distributed data structures like IMap, IQueue, MultiMap, ITopic etc... For complete list see the classes
35  * extending DistributedObject
36  * * Access to transactional distributed data structures like TransactionalMap, TransactionalQueue etc...
37  * * Ability to add cluster listeners to a cluster and entry/item listeners to distributed data structures.
38  * @see MembershipListener, IMap#add_entry_listener , IQueue#add_item_listener etc .
39  * * C++ Client is smart by default, which means that it knows where the data is and asks directly to correct node.
40  * Note that you can turn this feature off ( client_config#setSmart), if you don't want your clients to connect every
41  * node.
42  *
43  * Our C++ client is completely open source and the source code is freely available at https://github.com/hazelcast/hazelcast-cpp-client .
44  * Please feel free to contribute. You can join our community at https://groups.google.com/forum/#!forum/hazelcast where
45  * you can find answers to your questions.
46  */
